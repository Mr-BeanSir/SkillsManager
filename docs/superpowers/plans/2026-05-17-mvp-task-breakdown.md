# Skills Manager MVP Task Breakdown

> Status note (2026-05-18): this plan is complete through Task 15 and now serves as the MVP implementation baseline. Active implementation has moved to `docs/design/project-only-refactor.md`. Completed refactor work so far: Phase 1 Task 1.1 in `53f00d6 feat: add project-only schema migration` and Phase 1 Task 1.2 in `00932ae feat: add project-only migration service`.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement one task at a time. Each task must finish with tests/build evidence and a git commit before moving on.

**Goal:** Build the first working vertical slice of Skills Manager as a Tauri 2 desktop app with SQLite-backed state, symlink-aware domain services, and a minimalist management UI foundation.

**Architecture:** Tauri 2 app with React + TypeScript frontend and Rust backend. Rust owns SQLite, filesystem, symlink, app-data paths, and Tauri commands. The frontend renders a dense, warm monochrome desktop management UI and never talks to SQLite directly.

**Tech Stack:** Tauri 2, React, TypeScript, Vite, Rust, rusqlite or sqlx SQLite, Vitest, Rust unit tests, Phosphor/Radix-style icons, plain CSS.

**UI Constraints:** Use `minimalist-ui` and `web-design-guidelines`: warm monochrome palette, no gradients, no heavy shadows, 8px max ordinary card radius, semantic HTML, visible focus states, labelled controls, icon-only buttons with `aria-label`, paginated lists, dense table-first management layout.
